# Техническое задание на дизайн веб-приложения "Общий Сбор!"

## Общая информация о проекте

**Название:** Общий Сбор!
**Тип:** Веб-сервис для организации и поиска событий
**URL:** https://obschiysbor.vercel.app
**Целевая аудитория:** Активные люди 18-45 лет, ищущие события и компанию для досуга
**Платформы:** Desktop (1920x1080+), Tablet (768-1024px), Mobile (320-767px)

## Концепция и стиль

### Общая идея
Современный, дружелюбный и энергичный дизайн, который вдохновляет людей выходить из дома и участвовать в событиях. Должен быть:
- **Социальным** - акцент на людях и взаимодействии
- **Динамичным** - ощущение движения и активности
- **Доверительным** - чистый, профессиональный вид
- **Игровым** - элементы геймификации должны быть заметны, но не навязчивы

### Цветовая палитра
**Текущая (требует улучшения):**
- Primary: #0f4c81 (синий)
- Accent: #3498db (голубой)
- Background: #eef4ed (светлый)
- Card: #fafffe (белый)

**Рекомендации дизайнеру:**
- Создать более яркую и энергичную палитру
- Добавить акцентные цвета для разных категорий событий
- Предусмотреть тёмную тему (готова к внедрению)
- Использовать градиенты для создания глубины

### Типографика
**Текущая:**
- Font: system-ui, -apple-system, "Segoe UI"
- Размеры: 14px-36px

**Рекомендации:**
- Выбрать современный шрифт с хорошей кириллицей
- Чёткая иерархия заголовков (H1-H6)
- Читаемый текст для описаний событий
- Акцентный шрифт для брендинга

### Иконография
- Эмодзи используются как иконки категорий (🎲 🚴 🎨 и т.д.)
- Требуются custom иконки для системных действий
- Единый стиль иллюстраций для пустых состояний

Первое впечатление о сервисе, привлечение новых пользователей, быстрый старт для создания/поиска событий.

- Hero должен занимать 70-80vh
- Яркий фоновый градиент или паттерн

**Дизайн-требования:**
- Карточки с тенью и hover-эффектом (подъём вверх)
- Lazy loading изображений
- Свайп на мобильных
- Кнопка "Смотреть все" → /events

**Контент:**

#### 1.5 Секция "Статистика платформы"

- Светлый фон, выделяющийся от основного

## 2. Страница событий (Events) `/events`

### Назначение
Каталог всех событий с мощной системой фильтрации, карта, поиск.

### Блоки и элементы

#### 2.1 Заголовок страницы
**Контент:**
- Заголовок: "События" (H1)
- Кнопка "Создать событие" (primary, fixed справа сверху на desktop)

#### 2.2 Панель фильтров (левая колонка на desktop)
**Контент:**
- **Поиск** (input с иконкой 🔍)
- **Категории** (чекбоксы с 20+ категориями)
- **Дата события:**
  - Сегодня
  - Завтра
  - Эта неделя
  - Этот месяц
  - Выбрать диапазон (date picker)
- **Тип события:**
  - Онлайн
  - Офлайн
  - Все
- **Цена:**
  - Бесплатные
  - Платные
  - Все
- **Расстояние** (если геолокация разрешена):
  - Слайдер от 1 до 50 км
  - Текущая геолокация пользователя
- **Фильтр по полу** (если организатор указал):
  - Все
  - Мужчины
  - Женщины
  - Смешанное
- **Повторяющиеся события:**
  - Только разовые
  - Только повторяющиеся
  - Все
- Кнопка "Сбросить фильтры"

**Дизайн-требования:**
- Sticky панель при скролле
- На мобильных: выдвижное меню (drawer)
- Счётчики активных фильтров
- Анимация применения фильтров

#### 2.3 Переключатель режимов просмотра
**Контент:**
- Кнопки-табы:
  - "Список" (активна по умолчанию)
  - "Карта"

#### 2.4 Режим "Список"
**Контент:**
- Сортировка:
  - По дате (ближайшие)
  - По популярности (больше участников)
  - По расстоянию (если геолокация)
- Сетка карточек событий (как на главной, но подробнее)
- Пагинация или бесконечный скролл

**Дизайн карточки события:**
- Изображение 16:9 (или placeholder)
- Бейджи: категория, статус, цена
- Название события (H3)
- Краткое описание (2 строки, ellipsis)
- Организатор (аватар + имя)
- Дата и время
- Локация
- Участники (avatars stack + счётчик)
- Рейтинг (⭐ 4.5 / 10 отзывов)
- Кнопка "Подробнее"

#### 2.5 Режим "Карта"
**Контент:**
- Полноэкранная интерактивная карта (Яндекс.Карты)
- Маркеры событий (кластеризация при зуме)
- При клике на маркер → popup с mini-карточкой события
- Боковая панель с отфильтрованными событиями (скроллится синхронно с картой)

**Дизайн-требования:**
- Плавная анимация переключения режимов
- Цветовое кодирование маркеров по категориям
- Адаптивная высота карты

## 3. Страница детального просмотра события (EventDetails) `/events/:id`

### Назначение
Полная информация о событии, присоединение, чат, отзывы, карта локации.

### Блоки и элементы

#### 3.1 Hero-секция события
**Контент:**
- Изображение события (полная ширина, 16:9 на desktop, 4:3 на mobile)
- Overlay с основной информацией:
  - Бейджи: категория, статус (Upcoming/Ongoing/Completed/Cancelled)
  - Название события (H1)
  - Организатор (аватар, имя, кнопка "Профиль")

#### 3.2 Основная информация (левая колонка)
**Контент:**
- **Описание события** (markdown)
- **Детали:**
  - 📅 Дата начала и окончания
  - 🕐 Время
  - 📍 Локация (адрес + ссылка "Показать на карте")
  - 💰 Цена (0 ₽ или сумма)
  - 👥 Участники (5/20)
  - 🎭 Тип: Онлайн/Офлайн
  - 🚻 Фильтр по полу (если указан)

- **Специфичные данные для категорий:**
  - **Настольные игры:** список игр с превью
  - **Велопрогулки:** сложность, маршрут, снаряжение
  - **Походы:** дистанция, местность, что брать с собой

#### 3.3 Sidebar (правая колонка)
**Контент:**
- **Карточка организатора:**
  - Аватар (крупный)
  - Имя
  - Уровень (🏆 Уровень 15)
  - Созданных событий: N
  - Рейтинг: ⭐ 4.8
  - Кнопка "Написать"

- **Действия:**
  - Кнопка "Присоединиться" (primary, крупная)
    - Если уже участник: "Покинуть событие" (danger)
    - Если событие заполнено: "Событие заполнено" (disabled)
    - Если событие завершено: скрыта
  - Кнопка "Пожаловаться" (⚠️, вторичная, маленькая)
  - Кнопка "Поделиться" (📤, вторичная)
    - Dropdown: VK, Telegram, WhatsApp, Копировать ссылку
  - Кнопка "Экспорт в календарь" (📅, вторичная)
    - Dropdown: Google Calendar, .ics файл

- **Если пользователь - организатор:**
  - Кнопка "Редактировать" (🖊️)
  - Кнопка "Отменить событие" (🚫, danger)
  - Кнопка "Пригласить друзей" (✉️)

#### 3.4 Карта локации
**Контент:**
- Интерактивная карта (Яндекс.Карты)
- Маркер локации события
- Кнопка "Построить маршрут"

**Дизайн-требования:**
- Высота 400px на desktop, 300px на mobile
- Закруглённые углы
- Тень

#### 3.5 Список участников (сворачиваемый)
**Контент:**
- Заголовок: "Участники (5)" с иконкой ▶/▼
- При развороте:
  - Поиск по участникам (если участников много)
  - Сетка карточек участников:
    - Аватар
    - Имя
    - Бейдж "★ Организатор" (для создателя)
    - Дата присоединения
    - Город (если указан)
    - Кнопка "В друзья" (если не в друзьях)
    - Кнопка "Пригласить в событие" (только для организатора)

**Дизайн-требования:**
- Grid: 3-4 колонки на desktop, 2 на tablet, 1 на mobile
- Анимация разворачивания

#### 3.6 Чат события (только для участников)
**Контент:**
- Заголовок: "Чат события" (H3)
- Если не участник: "Чат доступен только участникам события"
- Если участник:
  - Область сообщений (скролл, max-height 500px)
  - Каждое сообщение:
    - Аватар отправителя
    - Имя
    - Текст сообщения
    - Время
  - Форма отправки:
    - Input (placeholder: "Введите сообщение...")
    - Кнопка "Отправить"

**Дизайн-требования:**
- Отличать свои сообщения от чужих (выравнивание справа)
- Real-time обновление (WebSocket)
- Автоскролл к новым сообщениям
- Уведомление о новом сообщении

#### 3.7 Отзывы и рейтинг (только для завершённых событий)
**Контент:**
- Заголовок: "Отзывы" (H3)
- Средний рейтинг: ⭐ 4.5 (15 отзывов)
- Если пользователь участвовал и не оставил отзыв:
  - Форма добавления отзыва:
    - Выбор звёзд (1-5)
    - Textarea (placeholder: "Поделитесь впечатлениями...")
    - Кнопка "Оставить отзыв"

- Список отзывов:
  - Аватар автора
  - Имя
  - Звёздный рейтинг
  - Текст отзыва
  - Дата публикации

**Дизайн-требования:**
- Интерактивные звёзды для выбора рейтинга
- Валидация: минимум 10 символов в отзыве
- Отзывы с фото автора

## 4. Страница создания события (CreateEvent) `/create-event`

### Назначение
Форма для создания нового события с подробными настройками.

### Блоки и элементы

#### 4.1 Заголовок
**Контент:**
- Заголовок: "Создать новое событие" (H1)
- Breadcrumbs: Главная / События / Создать событие

#### 4.2 Форма создания (многоколоночная на desktop)
**Поля:**

**Основная информация:**
- **Название события** (input, required)
  - Placeholder: "Например: Вечер настольных игр в центре города"
  - Max 100 символов
  - Счётчик символов

- **Описание** (textarea, required)
  - Placeholder: "Расскажите подробно о событии..."
  - Min 50, max 2000 символов
  - Markdown поддержка (жирный, курсив, списки)
  - Счётчик символов

- **Категория** (select/dropdown, required)
  - 20+ категорий с иконками

- **Изображение события** (image upload)
  - Drag & Drop зона
  - Превью загруженного изображения
  - Рекомендация: 1200x675px (16:9)
  - Max 5MB
  - Форматы: JPG, PNG, WebP

**Дата и время:**
- **Тип события** (radio buttons):
  - Разовое
  - Повторяющееся (раскрывает дополнительные поля)

- **Дата начала** (date picker, required)
- **Время начала** (time picker, required)
- **Дата окончания** (date picker, optional)
- **Время окончания** (time picker, optional)

- **Если "Повторяющееся":**
  - Частота повторения: (dropdown)
    - Ежедневно
    - Еженедельно
    - Ежемесячно
  - Дни недели (чекбоксы): Пн, Вт, Ср, Чт, Пт, Сб, Вс
  - До какой даты повторять (date picker)

**Локация:**
- **Тип события** (radio buttons):
  - Онлайн
  - Офлайн

- **Если "Офлайн":**
  - **Адрес** (input с автодополнением Яндекс.Карт, required)
  - **Выбор на карте** (интерактивная карта)
    - Маркер можно перетаскивать
    - Адрес автоматически обновляется

- **Если "Онлайн":**
  - **Ссылка на встречу** (input, required)
    - Placeholder: "https://zoom.us/j/..."
  - **Платформа** (select): Zoom, Google Meet, Skype, Discord, Другое

**Участники:**
- **Максимум участников** (number input, required)
  - Min 2, max 1000
  - Бесконечность (checkbox): "Без ограничений"

- **Фильтр по полу** (radio buttons):
  - Все
  - Только мужчины
  - Только женщины

**Монетизация:**
- **Цена участия** (number input)
  - По умолчанию: 0 ₽ (бесплатно)
  - Если > 0: показать информацию о комиссии сервиса
  - Промокод (input, optional):
    - Кнопка "Применить"
    - Информация о скидке после применения

**Специфичные поля для категорий:**

- **Настольные игры:**
  - Выбор игр из базы (мультиселект с поиском)
  - Кнопка "Добавить игру" (если нет в базе)

- **Велопрогулки:**
  - Сложность (radio): Лёгкая, Средняя, Высокая
  - Маршрут (textarea)
  - Необходимое снаряжение (textarea)

- **Походы:**
  - Дистанция (number input, км)
  - Тип местности (select): Лес, Горы, Равнина, Смешанный
  - Что взять с собой (textarea)

#### 4.3 Действия
**Контент:**
- Кнопка "Предпросмотр" (вторичная)
  - Показать модальное окно с превью события
- Кнопка "Создать событие" (primary, крупная)
- Кнопка "Отмена" (ссылка)

**Дизайн-требования:**
- Валидация в реальном времени
- Подсветка ошибок красным
- Сообщения об успехе/ошибке (toast notifications)
- Автосохранение в localStorage каждые 30 секунд
- Предупреждение при попытке покинуть страницу с несохранёнными данными

## 5. Страница профиля пользователя (Profile) `/profile`

### Назначение
Личный кабинет пользователя с редактированием профиля, событиями, друзьями, геймификацией.

### Блоки и элементы

#### 5.1 Шапка профиля
**Контент:**
- **Аватар** (крупный, 150x150px)
  - При наведении: "Изменить фото"
  - Upload с превью
- **Информация:**
  - Имя пользователя (H1)
  - Email (маленький текст)
  - Город (📍 если указан)
  - Уровень и прогресс:
    - 🏆 Уровень 15
    - Прогресс-бар до следующего уровня (XP: 1450/2000)
  - Участник с (дата регистрации)
- **Действия:**
  - Кнопка "Редактировать профиль" (если свой профиль)
  - Кнопка "Выйти" (красная, маленькая)

#### 5.2 Табы навигации
**Desktop:** Горизонтальные кнопки-табы
**Mobile:** Выпадающий список (select)

**Табы:**
1. Профиль
2. Приглашения (с бейджем количества)
3. Друзья

## Анимации и переходы

### Рекомендуемые анимации:
- **Появление элементов:** Fade in + Slide up
- **Переключение табов:** Fade + Slide horizontal
- **Hover карточек:** Scale up + Shadow increase
- **Модальные окна:** Fade in + Scale from center
- **Уведомления:** Slide in from right
- **Загрузка:** Skeleton screens или shimmer эффект
- **Прогресс-бары:** Smooth fill animation
- **Кнопки:** Ripple effect (Material Design)

### Timing:
- **Fast:** 150ms (hover, clicks)
- **Medium:** 300ms (transitions, fades)
- **Slow:** 500ms (page transitions)

### Easing:
- **ease-out** для появления элементов
- **ease-in** для исчезновения
- **ease-in-out** для transitions
